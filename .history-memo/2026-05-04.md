Шаг 4: Альтернатива — запустить на другом порте (если 5173 нужен Docker)
Если Docker-контейнер должен работать на 5173:
Не меняй vite.config.ts (запрещено по правилам проекта).
Вместо этого используй переменную окружения Vite:
powershell
Copy
$env:VITE_PORT=5174; npm run dev
Или в .env.local (в папке frontend):
plain
Copy
PORT=5174
Если Vite не подхватывает PORT, запусти напрямую:
powershell
Copy
npx vite --port 5174
Проверь: открой http://localhost:5174
Важно: Прокси в vite.config.ts настроен на /api → localhost:8000 — это не зависит от порта frontend, работает на любом порте Vite.
Формат ответа: Команда запуска + URL + скриншот/проверка загрузки страницы.

На основе этого анализа спроектируй светлую тему, которая:

Сохраняет иерархию контраста, роль каждого цвета и визуальную роль поверхностей.

Имеет комфортный фон (не чистый белый, а нейтральный светло‑серый или слегка тёплый/холодный оттенок) для уменьшения нагрузки на глаза.

Использует менее насыщенные акцентные цвета на светлом фоне, чтобы избежать «дрожания» и добиться хорошей читаемости.

Гарантирует доступность: контраст текста к фону не ниже WCAG AA для основных размеров шрифта.

Содержит «зеркальную» шкалу токенов: если в тёмной теме есть bg-surface-900, bg-surface-800, text-primary-100, то в светлой — bg-surface-100, bg-surface-200, text-primary-900 и т.п. (like‑for‑like токены с противоположного края палитры).

Опиши и зафиксируй дизайн‑токены для обеих тем в виде структуры, пригодной для кода (например, JSON или TypeScript‑enum/объект):

color.background.app

color.background.surface.[low|medium|high]

color.border.[subtle|strong]

color.text.[primary|secondary|muted|onAccent]

color.accent.[primary|success|warning|danger]

shadow.surface.[low|medium|high]

radius.* (если используется)

state.[hover|active|disabled] для кнопок, чипов и табов.

На основе токенов предложи конкретные CSS‑фрагменты (или Tailwind‑конфигурацию, или theme‑объект для используемой UI‑библиотеки), которыми можно:

Переключать тему на уровне корневого класса, например .theme-dark / .theme-light;

Обеспечить одинаковое поведение компонентов в обеих темах без дублирования логики (только подмена токенов).

Проверь и опиши:

Как в светлой теме будут выглядеть ключевые демонстрационные экраны:

дашборд с карточками метрик и графиком;

список/таблица документов;

модальные окна, уведомления, тосты.

Какие элементы нужно доработать отдельно (например, иконки с заливками, градиентные бейджи, фоны графиков).

Выведи в ответе:

краткое текстовое описание визуальных принципов светлой темы (в сравнении с тёмной);

таблицу токенов для dark / light (одна строка = один смысловой токен, два столбца с цветами);

пример кода с переключением тем (CSS‑переменные или theme‑объект).
