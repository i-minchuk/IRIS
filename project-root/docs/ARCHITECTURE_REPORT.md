## 🔍 Детальный анализ нарушений

### 1. Модуль `documents` импортирует `projects` и `variables`

**Вопрос:** Это обоснованная зависимость или нарушение?

**Варианты решения:**

#### Вариант A: Разрешить зависимость (если обосновано)
```python
# docs/modules/architecture.md
allowed_deps = {
    "documents": {"auth", "projects", "variables"},  # Разрешить
}
```

**Когда использовать:** Если документы действительно содержат информацию о проектах и переменных.

#### Вариант B: Вынести общую логику в отдельный модуль
```
backend/app/modules/
├── shared/           # ⬅️ Новый модуль
│   ├── project_ref.py
│   └── variable_ref.py
├── documents/        # Зависит от shared
├── projects/         # Зависит от shared
└── variables/        # Зависит от shared
```

**Когда использовать:** Если есть общая логика, используемая несколькими модулями.

#### Вариант C: Использовать события вместо прямых импортов
```python
# documents/service.py
from app.core.events import emit_event

class DocumentService:
    async def create(self, data):
        doc = await self.repo.create(data)
        emit_event("document.created", {"document_id": doc.id, "project_id": doc.project_id})
        return doc
```

**Когда использовать:** Если нужна слабая связанность.

### 2. Модуль `resources` импортирует слишком много

**Проблема:**
```python
resources/
├── time_tracking  # Зависимость
├── documents      # Зависимость
└── projects       # Зависимость
```

**Рекомендация:** Разделить `resources` на более мелкие модули:
```
resources/
├── file_storage/      # Работа с файлами
├── external_apis/     # Внешние API
└── cache/             # Кэширование
```
